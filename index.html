RDCO Biodiversity Checklist: Full-Stack Application & ETL Pipeline

The RDCO Biodiversity Checklist is a comprehensive, full-stack system designed to manage, validate, and present a database of approximately 8,000 recorded flora and fauna species within the Central Okanagan region. Developed for the Central Okanagan Naturalists' Club (CONC), the project bridges the gap between raw data management and public accessibility.

The project consists of two major components working in tandem: a robust Python ETL (Extract-Transform-Load) command-line utility that sanitizes and synchronizes a master Google Sheets dataset, and a dynamic PHP-driven web application that provides a structured, searchable interface for both general reference and expert ecological use.

System Architecture

Google Sheets (Master Dataset) → Python CLI ETL Tool (Validation & pandas) → SSH Tunnel → MariaDB (Production Database) → PHP Web Application (Data Presentation)

Web Application & Interface Features

The frontend provides an intuitive but deeply detailed way to explore biological classifications, built with a focus on usability and accurate data representation.

  • Taxonomic Navigation: A persistent sidebar allowing users to browse species by biological groups, sorted in ascending taxonomic order from primitive (Misc. Unicellulars, Algae) to complex (Birds, Mammals).
  • Advanced Search & Querying: A robust global search that parses scientific and common names, returning categorized results divided into "ID Help Results" (taxonomic rank breakdowns with reference texts) and "Taxa Results."
  • Aggregated Data Dashboards: A "Groups & Totals" view that calculates and displays the total number of species per biological group, categorized by their local observation status (Confirmed, Probable, To Be Determined).
  • Detailed Species Profiles: Individual taxon pages displaying full hierarchical taxonomy (Phylum, Class, Order, Family), RDCO Status, Naturalization, and local distribution notes.
  • Dynamic External Integration: Auto-generated contextual links to external ecological databases like iNaturalist and E-Fauna BC based on the current species profile.
  • Contextual User Feedback: General contact forms as well as taxon-specific "Submit a Correction" forms embedded directly on species pages to facilitate crowd-sourced database corrections from experts.

Data Pipeline & Integrity (Python ETL)

To ensure data consistency before it reaches the live website, I built a production-style CLI utility to handle the ingestion and synchronization of the master dataset. Designed for non-technical users, it provides a guided workflow with real-time progress indicators.

  • Data Extraction: Connects to the Google Sheets API via gspread to pull structured record updates.
  • Strict Validation Layer: Uses pandas to enforce primary key uniqueness, field character limits, schema consistency, and controlled vocabulary constraints (e.g., status codes, group types) before any database write operations. Custom exception handling provides precise feedback on validation failures.
  • Secure Database Operations: Establishes a secure SSH tunnel to the remote AWS server and performs an atomic database refresh using a truncate-and-replace strategy via SQLAlchemy.
  • Automated Backups: Triggers automated database backups following successful synchronization updates.

Tech Stack

  • Backend: PHP
  • Database: MariaDB / MySQL
  • Data Pipeline: Python, pandas, gspread (Google Sheets API), SQLAlchemy, sshtunnel, argparse
