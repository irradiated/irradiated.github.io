RDCO Biodiversity Checklist

The RDCO Biodiversity Checklist is a production data platform built to manage, validate, and present approximately 8,000 species records for the Central Okanagan region of British Columbia. Rather than a simple website, the project functions as a complete engineering system that transforms a continuously maintained ecological dataset into a reliable public-facing reference tool for researchers, naturalists, and regional users.

The system supports scientific-name and common-name search, hierarchical taxonomy browsing, species detail pages, biological status tracking, identification assistance ("ID Help"), and correction submission workflows. Each result page presents normalized taxonomy, RDCO confirmation status, naturalization data, external reference links, regional distribution notes, and domain-specific identification guidance.

The public interface is backed by a controlled ETL pipeline and relational database architecture that ensures taxonomic consistency, validation of controlled vocabularies, and safe operational updates by non-technical users.

System Architecture

Google Sheets (Master Dataset) → Python ETL & Validation Pipeline → MariaDB Relational Database → PHP Application Layer → Search + Taxonomy + ID Help Web Interface

The engineering goal is controlled transformation of semi-structured biological records into a normalized, searchable biodiversity knowledge base.

Data Ingestion & Validation Layer (Python ETL System)

The ETL pipeline acts as the system’s control plane. It governs how raw spreadsheet data is extracted, validated, transformed, and safely committed into the production MariaDB database.

Core Responsibilities

  • Connects to Google Sheets using gspread API integration
  • Processes and transforms ~8,000 structured biological records using pandas
  • Enforces primary key uniqueness and schema consistency
  • Validates field constraints, taxonomy groups, and controlled status codes
  • Prevents invalid writes through strict pre-commit validation gates
  • Performs secure remote database updates through SSH tunneling
  • Executes atomic refresh workflow using truncate-and-replace strategy

Operational Design

  • CLI execution modes for verify, update, statistics, and system help
  • Real-time status feedback and progress indicators for operators
  • Automated backup generation after successful synchronization
  • Designed for safe execution by non-technical staff through guided workflows
  • Custom exception handling for precise validation failure reporting

Application & Presentation Layer (PHP Web Platform)

The web application functions as the system’s query and presentation layer, translating normalized relational data into a searchable scientific reference interface.

Users can search taxonomic records by scientific or common name (for example, searching “wolf” returns both mammal and lichen records), browse major biological groups through a structured navigation system, and access dedicated species pages containing taxonomy, distribution records, RDCO status, native/exotic classification, external references, and identification guidance.

The interface also includes an ID Help subsystem for ambiguous taxa, allowing domain experts to provide taxonomic clarification beyond standard database records. A correction submission workflow supports ongoing community-driven maintenance and long-term data reliability.

Key Features

  • Search-driven species discovery using scientific and common names
  • Hierarchical browsing across major flora and fauna groups
  • Species detail pages with normalized taxonomy presentation
  • RDCO distribution tracking and confirmation status reporting
  • ID Help system for expert classification support
  • User correction and feedback submission workflow
  • Secure deployment with separation of secrets from source code
