RDCO Checklist – Full-Stack Data & Web Application

The RDCO Biodiversity Checklist is a data-driven biodiversity information system that provides a structured, searchable catalogue of approximately 8,000 flora, fauna, and fungi species within the Regional District of Central Okanagan.

The system is built around a centralized data pipeline where a master dataset is maintained in Google Sheets, processed through a Python ETL validation system, and stored in a MariaDB relational database. The web application layer (PHP) provides a public-facing interface for querying and displaying the processed dataset.

Rather than a static directory, the system functions as a continuously maintained data platform with enforced schema validation, ensuring consistency, integrity, and long-term maintainability of biological records.

System Architecture

Google Sheets (Master Dataset) → Python ETL Pipeline (Validation & Transformation) → MariaDB Database → PHP Web Application (Data Presentation)

Python ETL & Database Synchronization Pipeline (CLI Utility)

Developed a production-style ETL (Extract–Transform–Load) pipeline responsible for synchronizing a large biodiversity dataset between Google Sheets and a MariaDB database as part of the RDCO Checklist system. The tool processes a master dataset of ~8,000 species records and enforces strict data integrity rules before committing updates to the production database.

The system is implemented as a command-line utility with structured execution flow and real-time progress feedback. It connects to Google Sheets via API, performs data validation and transformation using pandas, and securely updates a remote MariaDB database over an SSH tunnel.

System Architecture

Google Sheets (Master Dataset) → Python CLI ETL Tool → Validation Layer (pandas) → SSH Tunnel → MariaDB Database

Core Responsibilities

  • Extracts structured data from Google Sheets using gspread API integration
  • Implements full data validation layer including:
    • Primary key uniqueness enforcement
    • Character limit constraints per field
    • Controlled vocabulary validation (status codes, group types)
    • Schema consistency checks across datasets
  • Transforms spreadsheet data into relational format using pandas
  • Securely connects to remote MariaDB database via SSH tunneling
  • Performs atomic database refresh using truncate-and-replace strategy
  • Provides CLI-based execution with real-time spinner and status feedback

Data Integrity & Validation System

The pipeline enforces strict domain-specific business rules before any database write operation, including schema validation, data type enforcement, and controlled taxonomy constraints. Custom exception handling is used to provide precise feedback on validation failures.

Operational Features

  • Command-line interface with structured execution modes (verify, statistics, update, help)
  • Real-time progress indicators for long-running operations
  • Automated database backup after successful updates
  • Safe database update workflow with pre-validation gatekeeping
  • Designed for non-technical users with a guided CLI workflow, reducing operational complexity and enabling safe execution of database updates
