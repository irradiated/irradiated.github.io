RDCO Checklist – Full-Stack Data & Web Application

A production-style data management and web presentation system for cataloguing over 8,000 species of flora and fauna in the Regional District of Central Okanagan (RDCO). The system was developed for the Central Okanagan Naturalists Club (CONC) and has been actively maintained since 2022.

The project is built as a complete data pipeline and web application ecosystem. A master dataset maintained in Google Sheets is processed through a custom Python ETL pipeline that cleans, validates, and enforces domain-specific business rules before inserting structured records into a MariaDB database.

A PHP-based web application then dynamically retrieves and presents the processed data from the database, allowing users to browse the catalogue and submit feedback.

System Architecture

Google Sheets (Master Dataset) → Python ETL Pipeline (Validation & Transformation) → MariaDB Database → PHP Web Application (Data Presentation)

Data Ingestion & Validation Pipeline (Python)

Developed a Python-based ETL (Extract–Transform–Load) pipeline as part of the RDCO Checklist system. A CLI utility that automates data ingestion from Google Sheets, applies validation and business rules using pandas, and loads sanitized data into a MariaDB database.

Overview

This system bridges spreadsheet-based data entry with a relational database backend, ensuring that incoming data is cleaned, validated, and structured before being persisted in SQL storage.

Key Responsibilities

  • Integrated with Google Sheets API to extract structured tabular data
  • Implemented data sanitization and transformation logic using pandas
  • Enforced business rules and validation constraints prior to database insertion
  • Mapped spreadsheet data into relational schema format
  • Automated loading of processed data into MariaDB

Tech Stack

  • Python
  • pandas
  • Google Sheets API (gspread)
  • MariaDB / SQL

System Role

This pipeline functions as the backend data processing layer for the RDCODChecklist ecosystem, ensuring that user-managed spreadsheet data is reliably transformed into structured database records.

Architecture Context
